Design choice

The server is very complex due to the design requirements. Introducing multi-processing was required in order to utilize as many of the CPU cores as possible as a server will have an unknown number of clients. In addition to the performance aspect, the non blocking aspect of handling client connections and logging required a multi-process or multi-threaded setup. In Python, the Global Interpreter lock causes multi-threaded code to run on a single core unlike other C based languages. I chose to go with a multi-process architecture for these reasons.
